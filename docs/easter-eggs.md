## Keyboard Triggers

Type these words anywhere on the page (no input field needed). Characters accumulate in a rolling 20-character buffer. Each fires once. All display in red (`log-red` class with saturated red color).

| Word | Response | Notes |
|------|----------|-------|
| `why` | `> why` / `good question.` | Permanent log entry |
| `help` | `> help` / `no one is coming.` | Permanent log entry |
| `quit` | `> quit` / `there is no quit command. there never was.` | Permanent log entry |
| `stop` | `> stop` / `the system does not recognise 'stop'.` | Permanent log entry |
| `hello` | `> hello` / `...hello.` | 1.5 s delay before appearing. Permanent |
| `god` | `> god` / `not here.` | Permanent log entry |

### Konami Code

Sequence: Up, Up, Down, Down, Left, Right, Left, Right, B, A

- Logs `I SEE YOU` in red
- Shows milestone banner `>>> you found it <<<` for 5 s
- Permanent log entry, fires once

## Visual Effects

### Counter Glitch (Phase 3+)

- **Trigger:** Phase 3 or later, not in Singularity, every 373rd tick with 15% probability
- **Effect:** Skills counter briefly displays `?` for 400 ms, then reverts
- **Recurring:** Yes, can happen multiple times

### Singularity: Future-Timestamped Log Entries

- **Trigger:** Singularity active, every 200th tick with 30% probability, before Void
- **Effect:** A log entry appears with a random timestamp up to 24 hours in the future. Messages rotate among:
  - `skill #[future count] deployed.`
  - `optimising.`
  - `no anomalies detected.`
  - `all systems nominal.`
  - `user presence: not required.`
- **Style:** dread
- **Recurring:** Yes, can happen multiple times

## Style Note

All easter egg log messages display in **red** (`#cc0000`) using the `logRed()` function and the `.log-red` CSS class with `filter: saturate(1000)`. This ensures red cuts through the B&W palette and survives late-game desaturation. There are no ghost/invisible messages — every egg is visible and permanent.

## Technical Notes

- All Easter egg state is stored in the `_e` object (in-memory only, not persisted to localStorage)
- Number triggers use a data-driven `_numEggs` object, checked in a loop within `checkEasterEggs()`
- Number triggers check within a window of `num` to `num + 10` to avoid missing fast-moving counters
- Special visual effects (666 red flicker, 42/8008/80085 counter changes) are handled by `_numSpecial()`
- Counter glitch (Phase 3+) fires on every 373rd tick with 15% probability
- Singularity future-timestamped logs fire every 200th tick with 30% probability
- Typed-word responses show `> word` then the response, both in red
- Interaction triggers (right-click, tab switch, resize, double-click) use DOM event listeners
- Red-styled messages use `logRed()` which applies the `.log-red` CSS class with `filter: saturate(1000)`
